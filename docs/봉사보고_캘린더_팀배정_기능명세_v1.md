# 봉사보고 + 캘린더 + 팀배정 기능명세서 v1

- 작성일: 2026-05-08
- 대상 서비스: `city-campaign-2026` (서울풍납 회중 2026 대도시 캠페인)
- 목적:  
  개인 봉사 보고를 표준화하고, 날짜 기반 배정/팀 운영을 관리자와 인도자가 한 화면에서 수행 가능하게 한다.

---

## 1) 목표

1. 사용자(일반/관리자/인도자)가 날짜 중심으로 봉사 일정을 확인한다.
2. 사용자별 봉사 보고를 손쉽게 작성/수정/조회한다.
3. 관리자가 날짜별/구역별 팀을 2~3명 단위로 유연하게 배정한다.
4. 비회원도 보고 제출 링크로 제출 가능하게 한다.
5. 관리자 화면에서 신청자/보고/팀배정을 통합 조회 및 관리한다.

---

## 2) 역할 정의

1. 일반 사용자 (`USER`)
- 본인 보고 작성/수정/조회
- 본인 일정 조회
- 본인 팀 배정 조회

2. 인도자 (`LEADER`)  
- 관리자와 유사한 조회 권한(읽기 중심)
- 담당 일정의 팀 배정 편집 가능(옵션)

3. 관리자 (`ADMIN`)
- 전체 데이터 조회/생성/수정/삭제
- 일정(캘린더 이벤트) 관리
- 팀 배정 관리
- 비회원 링크 발급/만료

4. 비회원 (`GUEST`)
- 링크 토큰으로 보고서 제출만 가능
- 제출 후 수정 불가(옵션: 1회 수정 허용)

---

## 3) 메뉴/화면 구조(IA)

1. 하단 메뉴
- 홈
- 보고
- 정보
- 마이

2. 신규/확장 화면
- `/report` 봉사 보고 작성/내역
- `/schedule` 월간 캘린더
- `/schedule/[date]` 날짜 상세(일정/팀배정 요약)
- `/schedule/[date]/[eventId]` 일정 상세 + 팀 명단
- `/admin/schedule` 일정 관리
- `/admin/assignments` 팀배정 관리
- `/admin/reports` 보고서 관리
- `/guest/report/[token]` 비회원 제출 페이지

---

## 4) 핵심 기능 명세

## 4.1 봉사 보고 작성

1. 기본 정보
- 봉사 참여일 (기본값: 선택 날짜)
- 봉사 유형 선택:
  - 회중봉사/순회구 봉사
  - 지하철 전시대 봉사

2. 회중봉사/순회구 봉사 입력 항목
- 재방문 (건수)
- 연락처(주소) 교환 (건수)
- 방문 요청 (건수)
- 성경 공부 제안 (건수)
- 성경 공부 시작 (건수)

3. 지하철 전시대 입력 항목
- 참여 시간대: 오전/오후 (단일 선택)
- 장소: 미사역/올림픽공원역/천호역(5)/천호역(6)/문정역 (단일 선택)
- 잡지 배부 수 (건수)
- 행누책 배부 수 (건수)
- 방문 요청 수 (회수)
- 유의점/기타사항 (텍스트)

4. 경험담
- 참여한 봉사 유형 (택1)
- 경험담 유형 (택1)
- 경험담 내용 (텍스트)

5. 저장 규칙
- 임시저장 가능
- 제출 시 숫자항목 0 허용, 텍스트 공란 허용
- 제출 후 본인 수정 가능 여부: 기본 `가능` (관리자 설정으로 마감 가능)

---

## 4.2 캘린더(월간/일간)

1. 월간 캘린더
- 캠페인 기간 헤더(예: 2026-05-01~2026-05-25)
- 날짜별 상태 색상:
  - 배정 있음
  - 보고 작성됨
  - 보고 미작성
- 날짜 클릭 시 상세 페이지 이동

2. 날짜 상세
- 당일 일정 목록(시간, 장소, 인도자/보조)
- 안내문(공지) 영역
- 팀명/동료 요약
- `자세히 보기` 클릭 시 팀배정 상세 이동

3. 일정 상세
- 구역 정보
- 팀별 명단 카드
- 인도자/보조 표시
- 관리자/인도자일 경우 `팀 편집` 버튼 노출

---

## 4.3 팀 배정(관리자/인도자)

1. 배정 단위
- Event(일정) 아래 Team(팀) 단위 생성
- Team 당 인원 2~3명 권장
- 1명/4명 이상 배정 시 경고(저장 차단 여부는 옵션)

2. 배정 방식
- 신청자 목록에서 선택 후 팀으로 이동
- 드래그앤드롭 또는 체크박스 일괄 이동
- 수동 입력(비회원/현장참여자)

3. 중복 방지
- 동일 날짜/시간에 한 사람의 다중 팀 배정 차단
- 충돌 시 경고 표시

4. 운영 보조
- 미배정자 자동 필터
- 팀 자동 균등 분배(옵션)

---

## 4.4 비회원 보고 링크

1. 링크 발급
- 관리자가 만료일/허용횟수 설정
- 토큰 URL 생성 후 복사

2. 제출 제한
- 토큰 만료 시 제출 불가
- 허용 횟수 초과 시 제출 불가

3. 보안
- 토큰은 난수 + 해시 저장
- PII 최소 수집 (이름/연락처 선택)

---

## 4.5 관리자 통합 화면

1. 보고서 관리
- 필터: 날짜/이름/봉사유형/제출상태
- 목록/상세/수정/삭제
- CSV 엑셀 내보내기(2차)

2. 신청자 관리
- 신청자 기본정보(회중, 성별, 출생연도, 요청사항)
- 배정 여부/배정팀 표시

3. 일정 관리
- 일정 추가/수정/삭제
- 인도자/보조 지정
- 안내문/집결 정보 입력

---

## 5) 데이터 모델(초안)

1. `MemberProfile`
- `id`, `name`, `congregation`, `gender`, `birthYear`, `phone?`, `note?`

2. `ScheduleEvent`
- `id`, `date`, `title`, `serviceType`, `startTime`, `endTime`, `location`, `guideText`, `leaderId?`, `assistantId?`

3. `Team`
- `id`, `eventId`, `name`, `zoneLabel?`, `sortOrder`

4. `TeamAssignment`
- `id`, `teamId`, `memberId`, `role`(member/leader/assistant), `isGuest`
- 유니크: (`eventId`, `memberId`) 논리 제약 필요

5. `ServiceReport`
- `id`, `reportDate`, `reporterId?`, `reporterName`, `isGuest`, `status`(draft/submitted)
- `stats`(JSON): 보고 건수 필드
- `subway`(JSON): 시간대/장소/배부수 등
- `experience`(JSON): 유형/내용
- `notes`

6. `GuestReportToken`
- `id`, `tokenHash`, `expiresAt`, `maxSubmissions`, `usedCount`, `isActive`

---

## 6) 권한 규칙

1. USER
- 본인 `ServiceReport`만 CRUD
- 배정 조회만 가능

2. LEADER
- 담당 `ScheduleEvent`와 `Team` 수정 가능
- 전체 보고서 조회(읽기) 가능 여부는 옵션

3. ADMIN
- 전체 CRUD 가능

4. GUEST
- 토큰 유효 시 보고서 `create`만 가능

---

## 7) 유효성/예외 처리

1. 숫자 필드: 음수 불가, 기본 0
2. 날짜: 캠페인 기간 밖 제출 시 경고 또는 차단(정책 결정)
3. 팀 인원: 2~3명 권장 경고
4. 중복 배정 차단
5. 저장 실패 시 사용자 메시지:
- 네트워크 오류
- 권한 없음
- 토큰 만료
- 중복 배정

---

## 8) 구현 단계 제안

1. 1단계(MVP)
- DB 스키마 추가
- 보고 작성/조회
- 월간 캘린더 + 날짜 상세
- 관리자 팀배정(기본)

2. 2단계
- 비회원 토큰 제출
- 신청자 관리표 UI
- 인도자 권한 분리

3. 3단계
- 자동 배정 보조
- CSV 내보내기
- 통계 대시보드

---

## 9) 수용 기준(acceptance criteria)

1. 사용자별로 날짜를 선택해 보고 제출 가능
2. 관리자에서 날짜 클릭 시 일정/팀/명단 확인 가능
3. 팀당 2~3명 배정이 가능하고 중복 배정 차단
4. 관리자에서 보고서 필터/상세 조회 가능
5. 모바일에서 캘린더/보고/상세 화면 사용 가능

---

## 10) 즉시 확정이 필요한 항목(의사결정)

1. `LEADER` 권한을 별도 롤로 둘지(`ADMIN` 하위권한으로 대체할지)
2. 보고서 제출 마감 정책(당일만/기간 내 자유)
3. 비회원 제출 시 수집할 최소 정보(이름 필수 여부)
4. 팀 인원 2~3명 규칙을 강제 차단할지, 경고만 할지
5. 신청자(게스트) 원본 표 데이터를 DB로 수동 입력할지 CSV 업로드 받을지

---
